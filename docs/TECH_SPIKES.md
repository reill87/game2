# 기술 스파이크 (구현 전 검증)

각 항목은 **최대 0.5~1일** 프로토타입으로 충분한지 판단.

## 1. 턴제 전투 UX (Phaser)

- [ ] 턴 순서 표시, 행동 선택, 로그 출력까지 **최소 UI**  
- [ ] 확정: 턴제 vs 실시간

## 2. 데이터 주도 스킬

- [ ] JSON 1개에서 스킬 3개 로드 → 전투에서 호출  
- [ ] `effect` 타입을 하드코딩 enum으로 막을지 결정

## 3. 저장 구조

- [ ] `localStorage` 단일 키 vs 버전별 마이그레이션 훅  
- [ ] 기존 `game` 세이브와 **호환 불가** — 사용자에게 혼동 없게 별도 키 (`devteam_rpg_v1`)

## 4. 콘텐츠 로딩

- [ ] Vite에서 `import mission from './missions/x.json'` vs 런타임 fetch

## 5. 성능 (모바일)

- [ ] 파티 3 + 적 4 스프라이트/도형 렌더링, 60fps 목표 여부

## 6. 기존 game에서 가져올 수 있는 것

- Pretendard, 해상도·스케일 설정 (`FIT`, 세로)  
- Supabase 연동 패턴 — **v2**까지 보류 가능  
- 직굑별 풍자 카피 스타일 — `messages` 패턴 참고

## 7. 스파이크 완료 정의

- 해당 체크박스에 **실험 브랜치 커밋 링크 또는 스크린샷** 1장 이상
