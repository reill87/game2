# 핵심 루프 (1페이지 초안)

## 세션 길이 목표

- **한 번 앉아서 3~10분**에 “한 사이클”이 완결되도록 설계 (이후 메타 진행은 저장)

## 루프 다이어그램 (텍스트)

```
[ 허브 / 팀 화면 ]
       ↓
  과제(던전) 선택  ← 난이도·보상·소요 시간 표시
       ↓
[ 과제 진행 ]  … 턴 또는 시간 단위로 “이슈 처리”
       ↓
  성공 → 보상 (XP / 골드 / 재료) → 스킬·장비·레벨 반영
  실패 → 페널티 (체력/사기 소모, 일부 보상 감소) → 허브
       ↓
[ 허브 ] 팀 스탯 확인, 장비 정리, 간단한 이벤트(선택)
       ↓
  (반복)
```

## 플레이어가 매 사이클 내리는 의사결정 (초안)

1. **누구를 데려갈지** (파티 2~3명, 직군 시너지)  
2. **이번 과제에서 무엇을 우선할지** (예: 버그 먼저 vs 기능 먼저 — v1에서는 단순 스킬 선택으로 대체 가능)  
3. **짧은 던전 vs 긴 던전** (시간 대 보상)

## 강화 클리커와의 차이

| 기존 (game) | RPG (game2) |
|-------------|-------------|
| 단일 “강화” 버튼 + 확률 | 과제 단위 **여러 턴** + 자원 관리 |
| 실패 = 단계 하락/소멸 중심 | 실패 = **전술·준비** 보완 후 재도전 유도 |
| 메시지 풍자가 핵심 보상 | **클리어·빌드** 자체가 보상 |

## 아직 결정 안 된 것 (체크리스트)

- [ ] 전투가 **완전 턴제**인지, **실시간에 가까운 액션**인지  
- [ ] 한 과제가 **1화면에서 끝**나는지, **미니 맵 이동**이 있는지  
- [ ] “보스”를 **릴리즈 데드라인**으로 고정할지, **P0 이슈 산**으로 할지
