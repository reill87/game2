# PRD — 개발팀 키우기 RPG (초안)

> 작성일: 2026-05-06  
> 성격: 취미 / 학습 / 완주 목표

## 1. 한 줄 정의

플레이어는 **기획·디자인·개발 직군으로 구성된 팀**을 육성하고, **스프린트·이슈·릴리스**를 은유한 **과제(던전)** 를 클리어하며 성장하는 **단기 세션형 RPG** (웹, 모바일 브라우저 대응).

## 2. 왜 만드는가

- 기존 `game`(강화 클리커)의 **핵심 재미가 강화 확률에만 묶여 있다**고 느껴, **의사결정·조합·진행감**이 있는 루프로 전환
- Phaser 또는 동일 스택으로 **씬 구조·데이터 주도 콘텐츠**를 한 번 더 연습

## 3. 목표 (성공 정의)

1. **v1 버티컬 슬라이스**를 `SCOPE.md`에 정의한 수준으로 플레이 가능하게 할 것  
2. **저장·이어하기** (localStorage 최소, 필요 시 이후 클라우드)  
3. 배포 URL 1개 (친구에게 링크로 공유 가능 수준)

## 4. 타겟 플랫폼

- 웹 (세로 화면 우선, 터치 + 키보드 보조)
- 네이티브 앱 비목표

## 5. 톤 & 세계관

- **직군 밈·회사 풍자**는 유지하되, **과도한 현실 브랜드/특정 회사** 직접 언급은 피함
- UI 카피는 가볍게, 전투/과제는 “티켓·버그·마감” 은유로 통일 (세부는 `SYSTEMS.md`)

## 6. 기술 스택 (초안)

- **게임**: Phaser 3 + TypeScript + Vite (기존과 동일 권장 — 학습 비용 절감)
- **데이터**: JSON 또는 TS 상수 모듈 → 나중에 스프레드시트 export 검토
- **백엔드**: v1 비목표 (리더보드·계정은 v2)

## 7. 문서 연결

- 루프·전투 규칙: `CORE_LOOP.md`, `SYSTEMS.md`  
- 범위 고정: `SCOPE.md`  
- 화면: `UI_SCENES.md`
