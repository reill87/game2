# 판교 IT 회사 키우기 — 컨셉 디자인 v0.1

> 게임의 차별화를 한국 IT 회사 시뮬 메타로 옮기는 피벗 기획서.
> v1 메커닉 검증을 마친 시점(2026-05-08)에서 시작. 코드와 함께 다듬어 가는 살아있는 문서.

---

## 1. 한 줄 정의

**"분당 셰어오피스에서 시작해 강남 사옥까지 — 한국 IT 회사 시뮬"**

플레이어는 5인 미만 IT 스타트업의 대표. 매주 자동으로 흐르는 시간 속에서 프로젝트 런칭 → 매출 → 사옥 이전 → 채용을 반복하며 회사를 키운다. 그 사이사이 단톡·조직개편·장애콜·임원 PT 같은 한국 IT 특유 모먼트가 끼어들어 결정을 강요한다.

기존 v1의 "인디 게임 스튜디오" 메타에서 이쪽으로 피벗. 본가 콘텐츠(『게임개발 스토리』 류)가 점유하지 못한 영역.

## 2. 차별화 3축 — "이 게임의 맛"

| 축 | 무엇 | 왜 |
|---|---|---|
| **사무실 진행 narrative** | 분당 → 판교 → 강남, 단계마다 시각·인원·문화가 바뀜 | 보상 루프의 가시화. "성장하는 회사를 보는 재미" |
| **한국 IT 직격 이벤트 풀** | 36+ 후보 / 24+ 구현 | "이거 우리 회사잖아" 모먼트. 본가가 절대 못 들어옴 |
| **직원 페르소나 + 진짜 퇴사** | 트레이트 메커닉 + 사기 30↓ N주 → 실제 퇴사 | 사기 메트릭에 진짜 무게. 야근 ON 전략의 진짜 비용 |

추가 후보(선택):
- **트렌드 메타** — 매 분기 G/T modifier 변동 (AI 봄 / 메타버스 침체 / Web3 거품 같은 시대 풍자)
- **사옥 단계별 일러스트** — 단계 진행의 시각 보상

## 3. 사무실 단계 — Narrative Arc

성장이 한 곳에서 정체되지 않게, 사옥 자체를 보상 루프의 핵심 축으로. 단계마다 시각·인원·문화가 가시적으로 달라진다.

| 단계 | 이름 | 인원 | 비용 | 주요 변화 | 시각 큐 |
|---|---|---|---|---|---|
| 1 | 분당 셰어오피스 | 3명 | 시작 | PM·디자이너·개발자 3인. 단톡 1개. | 좁은 데스크, 노트북 1대씩 |
| 2 | 판교 임대 사무실 | 4명 | 300g | QA 합류. 캡슐 커피머신. 회의실 1개. | 트인 오픈 스페이스 |
| 3 | 판교 자체 사옥 | 5명 | 800g | DevOps/그로스 1명. 옥상·라운지. | 자체 빌딩 일부 층 |
| 4 (v2) | 강남 사옥 | 6명 | 2000g | 신규 사업·해외 거점. | 강남 오피스 빌딩 |

각 단계 진입 = "**판교 입성**" 같은 짧은 컷 연출 + 회사 단톡에 축하 이모지 폭탄. 게임의 클라이맥스 중 하나.

v1 SCOPE는 1→2 한 단계만 구현. 이번 기획에선 3단계까지 설계 두고 코드는 점진적.

## 4. 슬롯·직군 매핑

기존 코드의 4 슬롯 구조 유지. `sound` 슬롯은 `qa`로 리네임.

| 슬롯 (코드) | 직군 표시명 | 비어 있을 때 |
|---|---|---|
| `planning` | 기획 / **PM** | 진행도 ↓ (스펙 모호) |
| `graphics` | 디자인 / **UI·UX 디자이너** | Appeal ↓ (시안 부족) |
| `qa` *(기존 sound)* | **QA** | BugDebt 가속 (테스트 늦음) |
| `programming` | 개발 / **개발자** | Progress 정체 |

QA가 1단계에선 비어 있고 2단계에서 합류 — "스타트업 초창기 QA 늦게 합류" 패턴 그대로. 도메인 매칭 로직은 기존 정/오배치 그대로.

| 코드 | 라벨 | 특성 |

## 7. K-X 메메 카탈로그 — 풍자 패턴 뱅크

한국 IT 종사자 대부분이 공유하는 메타-농담. 본가 콘텐츠 카피 걱정 없이 마음껏 가져갈 수 있는 산업 공유 자산. **이벤트의 절반 이상을 이 패턴으로 채울 수 있다.**

### "K-X = X를 빠르게 / 형식만"

| K-X | 실체 |
|---|---|
| K-에자일 | 워터폴 빠르게 + 데일리 스탠드업 |
| K-OKR | 분기마다 새로 쓰는 KPI |
| K-스크럼 | 매일 회의 30분 + 스프린트는 분기에 한 번 |
| K-수평조직 | 호칭만 "ㅇㅇ님" / 결정은 위에서 |
| K-DevOps | 개발자가 운영도 함 |
| K-MSA | 모놀리스를 여러 작은 모놀리스로 |
| K-CI/CD | CI는 자동, CD는 사람이 누름 |
| K-자율좌석 | 어차피 자기 자리 똑같음 |
| K-재택근무 | 9시 슬랙 온라인 + 노트북 항상 켜둠 |
| K-1:1 미팅 | 이번주 진척 보고 |
| K-360도 평가 | 누가 누구한테 했는지 다 앎 |
| K-MBO | 목표는 부장이 정해 줌 |
| K-코드리뷰 | "👍" 한 글자 |
| K-TDD | 테스트는 마지막 주에 |
| K-혁신 | 또 조직 개편 |
| K-DX | TF 만들기 |
| K-사일로 해체 | 부서 이름 바꾸기 |
| K-PoC | 임원이 보고 싶어해서 만든 데모 |
| K-온보딩 | 첫날 노트북 + "환영합니다" → 알아서 |

### "X 라고 적혀 있는데 실제로는 Y"

| 적힌 것 | 실제 |
|---|---|
| 자율 | 알아서 야근 |
| 워라밸 | 워크 = 라이프 |
| 강제 아닙니다 | 강제 |
| 캐주얼 데이 | 어차피 매일 |
| 자기계발비 | 사용 시 "어디 가셨어요?" |
| 사이드프로젝트 OK | 본업 끝나고 |
| 휴가 자유 사용 | 부서 분위기 봐 가며 |
| "고견 부탁드립니다" | 답정너 |
| "잠깐 시간 되세요?" | 1시간 회의 |
| "이번 분기는 부채 청산" | 다음 분기에도 부채 청산 선언 |

### 회의·결정의 K 모먼트

- "결정은 미팅 끝나고 따로"
- 임원 PT = 슬라이드 100장
- 그룹콜 = 한 명이 30분 말함
- "확인하셨나요?" 압박
- 새벽 슬랙 = 결정해 놓고 알림
- 단톡·슬랙·이메일에 같은 내용 3번
- "결재 라인 줄이자" → 라인 추가됨

### 일정·릴리스의 K 모먼트

- 프리징 직전 스코프 추가
- 점심·금요일 배포 금기 (어겼다가 장애)
- D-1 일주일 야근
- "이번 분기는 안정화" → 다음 분기에도 안정화
- 장애 컨콜 = 진단 30분 / 책임 30분 / 복구 5분
- 출시 직후 트위터 검색
- 출시 후 대기조

### 도구·기술의 K 모먼트

- 노션 쓰는데 첨부는 카톡
- 슬랙 두고 진짜 일은 카톡
- Figma 라이선스 1개 (돌려 씀)
- 레거시 주석 = "왜 이런지 모름"
- git blame = 본인
- ChatGPT 도입 → "이거 검증된 거?"
- 새 프레임워크 → "왜 굳이?"

### 직급·평가의 K 모먼트

- 호칭만 "ㅇㅇ님" / 결정은 동일
- 분기 평가 = 부장이 점수 매김
- 면접관 차출 = 자기 일도 못 함
- "임원 의견 = 회의 결론"
- 신임 임원 = 모든 거 reset
- 시니어 = 다 함
- 인턴 = 정직원 일

## 9. 직원 페르소나 (Slice P-3 디자인)

이름 + 트레이트 1-2개. 트레이트는 일부 이벤트 트리거 / 효과를 변동.

| 직원 | 직군 | 트레이트 | 작용 |

## 11. 자료 조사 가이드

### 원칙

> 사용 가능한 것은 **트로프(산업 일반 현상)**이고, **특정 작품의 표현은 그 작품의 것**.

휴리스틱: **두 곳 이상에서 발견되는 같은 패턴 = 산업 공유 자산** → 안전하게 트로프 뱅크에 추가.
한 작품에서만 발견되는 표현 → 손대지 않음.

### 자료 카테고리별 활용법

| 유형 | 사용 방식 |
|---|---|
| **1차 소스 (가장 안전)** | 본인 회사 경험, 동료·전 동료 인터뷰. 30분만 친구 둘 만나도 카탈로그 절반 채워짐. 저작권 무관. |
| **공개 통계·기사** | 노동·근무환경 통계, 산업 보고서, 일반 매체 IT 기사. 사실은 저작권 보호 X — 인용·요약 OK. |
| **온라인 커뮤니티** | OKKY·블라인드·디시 IT·잡플래닛 등 — **트렌드 파악용**. 개별 글 옮기기 X, 공통 패턴만 추출. |
| **개발자 매체·뉴스레터** | 회사 문화 다룬 칼럼 — 산업 현상 식별 도움. 인용은 짧게 출처. 카피는 본인이. |
| **웹툰·영상 (영감만)** | 보면서 산업 일반 패턴인 것 vs 작가 창작인 것 구분. 전자만 트로프 뱅크. 후자는 손대지 않음. |
| **컨퍼런스·팟캐스트** | DEVIEW·FE Conf 등 조직 문화 트랙 발표 — 사례 공개. 사실 정보로 활용. |

### 추출 워크플로우

1. 카테고리 잡고 시작 (위 6 카테고리 그대로)
2. 본인 경험 30분 자유 작성
3. 친구 2-3명 짧은 대화 ("최근에 황당했던 모먼트?")
4. 공개 자료 검색 — 산업 일반 현상으로 굳어진 것만
5. 트로프 뱅크에 정리
6. 게임 카피는 본인이 새로 작성

### 안전 라인

- 특정 회사·서비스·인물 이름 직접 사용 X
- 메신저는 "단톡"·"슬랙" 일반어. 카카오톡 brand 직격 X
- 직군 비하·인격 공격 X — 자조와 공감의 결로
- 게임 크레딧에 "각종 종사자분들과의 대화·공개 자료 참고" 명시

## 12. 트로프 뱅크 형식

노션 데이터베이스 / 스프레드시트 한 장에 다음 컬럼:

| 트로프 | 카테고리 | K-X 형식 가능? | 게임 이벤트 후보? | 비용/효과 아이디어 | 출처 |
|---|---|---|---|---|---|
| 워터폴 빠르게 = 에자일 | 도구·기술 | ✅ | ✅ | -50g, 효율 +5% | 본인 경험 |
| 호칭만 ㅇㅇ님 | 조직·문화 | ✅ | ✅ | 사기 ±0 농담 | 일반화 가능 |
| 새벽 P0 단톡 | 메신저 | ❌ | ✅ | stamina -8, BugDebt -10 | 본인 + 동료 |
| 분기마다 조직개편 | HR | ❌ | ✅ | 모두 사기 -5 | 일반화 가능 |
| ... | ... | ... | ... | ... | ... |

본인 경험 + 친구 2-3명만으로도 30-50개 채워지는 게 보통. 다 채우면 그걸 기준으로 이벤트 24개를 자기 카피로 작성하면 차별화가 그대로 묻어 나옴.
